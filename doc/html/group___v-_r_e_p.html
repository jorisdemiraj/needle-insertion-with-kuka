Enumeration Type Documentation

enum ObjectType

Each Object in the scene must be assigned one of the following types.

Enumerator
SHAPE 

Arbitrary 3D mesh object.

CONSTRAINT_POINT 

Constraint that holds the haptic device at a given point. The point is defined by State::Pos.

CONSTRAINT_SEGMENT 

Constraint that holds the haptic device on a given segment. The segment is defined by a point (State::Pos) and a vector (State::Dir).

CONSTRAINT_PLANE 

Constraint that holds the haptic device on a given plane. The plane is defined by a point (State::Pos) and a normal direction (State::Dir).

void hapticLoop ( )

The haptic loop runs in the background (with high priority) and renders all Objects haptically to all devices in the scene. Use hapticStart() to start the haptic loop, and hapticReset() to stop it.

Note
The hapticLoop is started and stopped automatically by hapticConnect() and hapticDisconnect(), according to the number of devices connected.

float  toolRadius,
float  workspaceRadius 
)

\brief Connect to a haptic device and add it to the scene.

If successful, the device is added to the device list and will be controlled by the hapticLoop().

Parameters
deviceIndexindex of the device to open in the list of devices detected by the operating system. This is guaranteed to be the same between executions as long as no hardware changes occur.
toolRadiusradius of the haptic device proxy (tool) in the simulated scene, expressed in m.
workspaceRadiusradius of the simulated workspace that the physical device workspace must be scaled to, expressed in m.
Returns
true on success, false otherwise.
